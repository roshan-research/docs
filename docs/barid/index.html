    این ای‌پی‌آی برای خلاصه‌سازی اسنادی که از قبل در ایندکس‌های ذخیره‌سازی قرار دارند استفاده می‌شود.

    پارامترها: id (لیست): شناسه‌های اسناد موردنظر. index (رشته): ایندکس محل ذخیره‌سازی اسناد. type (رشته): نوع سند (مثلاً "نامه"). content_key (رشته): کلید محتوای اصلی (مثلاً "body"). type (رشته): نوع خلاصه‌سازی. مقادیر قابل قبول: word, conversation, bullet. level (رشته): سطح خلاصه‌سازی. مقادیر قابل قبول: level_1, level_2, level_3.

    POST /barid/summarize/

    این ای‌پی‌آی برای پاسخ به سوالات بر پایه‌ی اسنادی که از قبل در ایندکس مشخصی ذخیره شده‌اند استفاده می‌شود. با ارسال شناسه سند (یا چند سند)، ایندکس مربوطه، نوع سند و لیستی از سوالات، سیستم پاسخ‌های متنی را از دل محتوای اسناد استخراج می‌کند.

    پارامترها: id (لیست رشته): لیستی از شناسه‌های سند. index (رشته): نام ایندکس محل ذخیره‌سازی اسناد. type (رشته): نوع سند (مثلاً "نامه"). content_key (رشته): کلید محتوای مورد تحلیل (مانند "body"). questions: لیستی از سوالات متنی که باید به آن‌ها پاسخ داده شود. هر سوال یک آبجکت است که دارای دو کلید text و type است. مقدار text همان سوال است و مقدار type نوع سوال را مشخص میکند که در حالت پیش فرض str است. این مقدار اختیاری است.
